# Information Clipper（信息剪报）

把散落在各处的网页内容，像剪报纸条一样存到本地，再也不用依赖某个 App 的收藏夹了。

---

## 核心理念

这玩意儿说起来也简单——就是"剪报"。

学生时代老师教过一种方法：把试卷和练习册的错题剪下来贴到本子上，减少无效抄写，把精力放在记解题思路。当年有些老师管这叫"邪修派"，但用过的人都知道，确实比手抄题目省事太多。

信息剪报就是这个思路的 AI 版本：

不用自己动手整理，只需要把觉得有意思的链接扔给 AI，它会用剪刀咔嚓咔嚓把好东西剪下来，贴成一个结构完整的 Markdown 文件，存进你的本地知识库。

---

## 工作原理

```
发送链接 → AI 自动抓取 → 整理为 Markdown → 存入 Raw 文件夹
```

具体来说，AI 会做这几件事：

1. 识别链接来自哪个平台（微信公众号、即刻、知乎、Bilibili 等）
2. 绕过 SPA 限制直接抓取正文（urllib 发请求，不需要浏览器）
3. 清洗各种垃圾：HTML 注释、JS 变量、CSS 残留，一个不留
4. 提取元数据：标题、作者、发布时间、原文链接
5. 自己理解文章后写一段摘要（不是简单截取前几百字）
6. 统计关键词（词频分析 + 标题优先）
7. 自动加上 ## 分节，保留原文的列表和引用块

---

## 输出格式

每个剪报是一个完整的 Markdown 文件，长这样：

```yaml
---
title: "登味"是什么味
type: raw
maturity: draft
date: 2026-04-23
source_platform: 微信公众号
original_url: https://mp.weixin.qq.com/s/xxx
recorded_at: 2026-04-23 15:10:51
summary: 这篇文章探讨了"登味"梗从贬义到自嘲的演变……
keywords: [老登, 小登, 登味, 代际, 自嘲]
tags: [网络流行语, 社会观察]
---

{正文内容，有结构，有分节}

## 变更记录

- 2026-04-23 15:10:51 — 初始创建
```

---

## 支持的平台

- 微信公众号 — SPA，需从 og:title 取文章标题
- 即刻 — 移动端 m.okjike.com，数据嵌在 JSON 里
- Bilibili — 视频/专栏均支持
- 知乎
- 小红书
- GitHub — 用 API 拿元数据，raw 获取 README
- 通用网页 — 抓 `<title>` 和 `<article>` 区域

---

## 增量更新

同一个链接再次被抓取时，不会替换原文，而是把新内容追加到变更记录里：

```markdown
## 变更记录

### 2026-04-25 10:30

#### 新增内容
{新内容原文照录}

#### 内容对比摘要
评论区新增 3 条评论，补充了架构图说明
```

这样随时可以追溯历史版本，知道原文是什么时间因为什么原因变了。

---

## 在 Wiki-Compiler 中的位置

```
知识库/
├── 1.raw/                    ← 剪报 raw 层（Information Clipper 维护）
│   ├── 2026.04.23-标题.md
│   └── ...
├── 2.wiki/                   ← 编译输出层（Wiki-Compiler 编译）
│   └── ...
└── 3.outputs/                ← 最终产出
    └── ...
```

Information Clipper 只做信息收集这一步。raw 文件可以单独用，也可以交给 Wiki-Compiler 编译成长青知识库。

## 设计哲学

就一个字 — **爽**。不只追求有用，不只追求体系，"这个挺有意思"就随手存一下。不要求每篇都系统有用，更注重趣味性和个人启发。

就像你的 AI 助理用剪刀咔嚓咔嚓把好东西都剪下来贴给你一样~ ✂️
